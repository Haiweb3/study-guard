# StudyGuard

学习/工作时屏蔽微信、QQ 等聊天软件干扰的 macOS 菜单栏小工具。

## 功能

- 两种屏蔽模式：**强制退出**（kill 黑名单 app，期间会持续巡检防止重新打开）/ **仅静音**（只屏蔽通知，app 仍能打开）
- 手动结束 或 定时自动结束
- 黑名单可自定义增删（默认预填微信、QQ）
- 提前结束需要 5 秒倒计时确认，防止一时偷懒
- 每次学习记录会存档，菜单栏面板里能看到最近的专注历史
- 全局快捷键 **⌃⌥S**（Control+Option+S）可以随时呼出/收起面板，不依赖菜单栏图标是否可见

## 开发 / 构建

```bash
export PATH="$PATH:$(go env GOPATH)/bin"   # 确保 wails3 在 PATH 里

wails3 dev              # 开发模式，热重载
wails3 build            # 编译出 bin/study-guard
wails3 task darwin:package   # 打包成正式的 bin/study-guard.app（带签名、图标）
```

日常使用直接打开 `bin/study-guard.app` 即可，建议拖到「应用程序」目录，或加入登录项自动启动。

## “仅静音模式”需要的一次性设置

macOS 没有公开 API 让第三方程序直接创建/编辑「专注模式」的通知过滤规则，所以仅静音模式需要你先手动设置一次，之后 StudyGuard 就能用「快捷指令」帮你一键开关：

1. **创建专注模式**：系统设置 → 专注模式 → 点 `+` 新建一个，命名为 **学习**。
   在它的通知设置里，把微信、QQ（以及你黑名单里的其他 app）都设为「不允许通知」。
2. **创建两个快捷指令**：打开「快捷指令」App，新建两个快捷指令：
   - 名字必须是 **`学习模式-开`**：动作选「设定焦点模式」→ 选「学习」→ 「打开」。
   - 名字必须是 **`学习模式-关`**：动作选「设定焦点模式」→ 选「学习」→ 「关闭」。
   （名字必须完全一致，StudyGuard 是靠名字调用 `shortcuts run` 的。）
3. 设置好之后，在 StudyGuard 面板里选「仅静音」模式开始学习模式，会自动执行对应的快捷指令。

如果没做这一步，选「仅静音」开始时会报错（提示找不到快捷指令），强制退出模式不受影响，随时可用。

## 数据存储

配置和历史记录都是 JSON 文件，存在：

```
~/Library/Application Support/StudyGuard/config.json    黑名单配置
~/Library/Application Support/StudyGuard/sessions.json  历史记录
```

直接删除这两个文件即可清空所有数据，恢复默认黑名单（微信、QQ）。

## 已知问题

- 在菜单栏图标已经很拥挤、且屏幕有摄像头缺口（notch）的机型上，新加入的菜单栏图标有可能被挤到缺口区域而不可见。这是系统菜单栏空间不足导致的，不是 StudyGuard 本身的问题；遇到时可以先退出几个不常用的菜单栏小工具腾出空间，或使用 Ice / Bartender 等菜单栏管理工具整理图标，也可以直接用下面的全局快捷键代替点图标。

## 全局快捷键需要的权限

⌃⌥S 是靠监听键盘事件实现的，macOS 要求授权才能监听全局键盘事件：

1. 第一次按快捷键如果没反应，去 **系统设置 → 隐私与安全性 → 辅助功能**，把 `study-guard` 加进去并打开开关（如果列表里没有，需要先按一次快捷键触发系统弹出请求，或手动点 `+` 把 `/Applications/study-guard.app` 加进去）。
2. 授权后**需要重新打开一次 StudyGuard**（退出再启动），快捷键才会生效。
3. 如果不想授权这个权限，菜单栏图标点击的方式依然完全可用，互不影响。
